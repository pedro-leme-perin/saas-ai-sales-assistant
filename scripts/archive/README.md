# scripts/archive/

One-shot deploy PS1 scripts from past sessions (S63 → S67-B).

## Purpose

These PowerShell wrapper scripts were generated by the agent during
sequential commit + push cycles (Pedro-side bypass for sandbox-Linux
limitations on Windows mount: pnpm install, git push credentials).

Each script:

- Stages specific files for one session (e.g. S66-A coverage ratchet)
- Writes commit message via `Out-File UTF-8` (lesson S62: PS heredoc
  embedded quotes break)
- Calls `git commit -F` + `git push origin main`
- Some include extra steps (sanitize `.git/config` NUL bytes, restore
  corrupted working tree files)

After the session merged, the script is one-shot and serves only
historical / forensic purposes.

## When to consult

- Reproduce a past deploy step-by-step
- Understand sandbox bypass strategies for Windows mount issues
- Archeology of S65 carryover roadmap (pre-commit + coverage + lint)

## When NOT to use

- Do NOT re-run any of these scripts on the current state — most assume
  files already created by sandbox in a specific transient state. Running
  them out-of-context will fail or do harm.
