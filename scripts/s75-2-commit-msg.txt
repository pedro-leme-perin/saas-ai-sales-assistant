fix(security): bump lodash to ^4.18.0 via pnpm overrides (S75-2)

Mitigates CVE-2026-4800 — high-severity RCE via prototype pollution in
`_.template` when user-controlled input reaches the template compiler
(CVSS 8.1).

Vector: transitive via @nestjs/config + @nestjs/swagger + tooling
(eslint, jest, etc). _.template rarely used in prod, but presence of
the gadget enables RCE if input traverses parsing.

Strategy:
- pnpm.overrides entry `lodash: ^4.18.0` (same-major range; lodash 4.x
  is long-stable line ~7 years without 5.x published — zero risk of
  silent major bump despite using ^ instead of ~).
- 1-commit-per-package per lesson #17; validated post-S75-1 green
  (CI #292 end-to-end).
