Recovery commit for S75-4. Previous commit 8a7c2f3 was created
accidentally by re-execution of s75-3-commit.bat (Return key on selected
file in File Explorer), which:
- Used the S75-3 commit message template (wrong: should be S75-4)
- Omitted root package.json from `git add` list (the file was modified
  with follow-redirects override but not staged because the S75-3
  commit script had a hardcoded list excluding root package.json)

  not matched to staged package.json — causing CI #298 Install job
  failure with --frozen-lockfile mismatch.

This fixup:
- Adds root package.json with `follow-redirects: ~1.16.0` to
  pnpm.overrides (alphabetic between @clerk/shared@3 and lodash).
- Regenerates pnpm-lock.yaml to match.
- Restores lockfile/package.json consistency for CI install gate.

The S75-4 docs (CHANGELOG v0.75.4, PROJECT_HISTORY S75-4 section,
CLAUDE.md §2.1 S75-4 row) were correctly committed in 8a7c2f3 even
